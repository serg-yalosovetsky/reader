# ADR Log

Решения, которые не стоит менять без веской причины. Самые свежие — внизу.

## ADR-001. foliate-js как движок рендера (EPUB + FB2)
Нужны EPUB и **FB2** (рус-фанфики часто FB2). epub.js не умеет FB2. foliate-js (MIT)
умеет оба, рендерит на клиенте (сервер не тратит ресурсы), даёт CFI и fraction для
прогресса. Завендорен (нет npm-релиза). **Не** заменять на epub.js.

## ADR-002. FanFicFare + собственные адаптеры
FanFicFare покрывает ficbook/fanfics.me/AO3/ffn из коробки и запускается subprocess'ом
(изоляция памяти). Для сайтов без адаптера — свои: **author.today** (XOR-расшифровка
глав), **readli.net**, **searchfloor.org**. Не переписывать FanFicFare-часть вручную.

## ADR-003. SQLite + WAL, не Postgres
RAM-дисциплина VPS. SQLite + **WAL + busy_timeout** достаточно для одного пользователя
и устраняет `database is locked` при долгом мониторинге + параллельных запросах.
Файл-БД в `data/`. Не вводить серверную СУБД без явной нужды.

## ADR-004. Бэкенд на 127.0.0.1, наружу — nginx + существующий SSO
Прод-VPS. Доступ наружу только по HTTPS через nginx с уже существующим vps-sso
(Clerk), как у других проектов. Бэкенд не слушает публичный интерфейс. Новый порт
наружу — согласовывать с человеком.

## ADR-005. Креды — Fernet локально, без Bitwarden
Рассматривался Bitwarden, но он требует хранить мастер-секрет на VPS — это более
мощный секрет, чем сами пароли сайтов. Решено: пароли/cookies аккаунтов шифруются
Fernet (`crypto.py`, ключ `data/secret.key`), живут только в нашей БД.

## ADR-006. Sync с ReadEra — через РУЧНОЙ бэкап, не авто-синк Premium
ReadEra Premium синхронит в Google Drive только КНИГИ (`ReadEra/Books/`), а прогресс —
в скрытый appDataFolder (снаружи недоступен). Поэтому прогресс берём из **ручного
бэкапа** ReadEra (`.bak` = zip с `library.json`), который ходит через папку Drive
(rclone). Матч книг — по `doc_sha1`. Запись назад — патч `.bak` + ручной restore
(best-effort; авто-вливания у ReadEra нет).

## ADR-007. Идентичность книги = SHA-1 файла; имя в хранилище = <sha1>.<ext>
Чтобы прогресс совпадал между читалкой, Calibre и ReadEra, везде используется один и
тот же файл (один SHA-1). Скачанные книги кладём в `ReadEra/Books` — Premium тянет их
на телефон, и `doc_sha1` совпадает с нашим.

## ADR-008. ficbook — только через cloudscraper
ficbook закрыт DDoS-Guard: с дата-центрового IP VPS httpx/requests получают
«Проверка безопасности» (403). cloudscraper пробивает челлендж. Применяется и в
фиде (`feeds._ficbook_feed`), и в скачивании (`-o use_cloudscraper=true`), и при
скачивании обложки. author.today/fanfics/readli/searchfloor с VPS доступны обычным клиентом.

## ADR-009. Платная author.today → фоллбэк на бесплатный полный текст
Платная AT-книга анонимно отдаёт только превью (1 глава). Детект «нет Свободного
доступа» → `PaidContentError` → `chain._fallback_free` ищет книгу по названию на
**searchfloor** (прямое скачивание `/book/<id>` = полный FB2), затем readli.
Цель — «всегда вся книга».

## ADR-010. Дедуп по (название+автор), хранить самый полный файл
Одна книга из разных источников/прогонов не должна плодить карточки.
`services.register_download` дедупит по source_url ИЛИ нормализованным (title, author);
при совпадении оставляем файл большего размера (полная версия). Чистка существующих —
`POST /api/library/maintenance`.

## ADR-011. Обложку встраиваем в файл книги
Обложка должна быть и в Calibre, и в ReadEra, не только в нашей сетке. Поэтому при
скачивании обложка (og:image источника или встроенная) встраивается в EPUB
(`set_cover` / FanFicFare `include_images`); FB2 уже несёт обложку. `cover_path` в БД —
для быстрого показа в библиотеке.

## ADR-012. author.today — вход с подтверждением устройства
Вход с нового устройства (IP VPS) требует email-код. Подтверждается разово
(код читается из Gmail, отправляется в той же сессии) → устройство становится
доверенным, дальше вход без кода. Поддержка cookie-сессии в `Account.cookies` есть
как запасной механизм.
