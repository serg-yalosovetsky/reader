# План: озвучивание книги с подсветкой текущего слова (TTS)

Цель: кнопка «Слушать» в читалке — голос читает текст, текущее слово подсвечивается,
страница/лента следуют за чтением. Качественный русский голос.

## Выбор движка

**edge-tts (Microsoft Edge TTS)** — бесплатный, отличные русские голоса
(`ru-RU-SvetlanaNeural`, `ru-RU-DmitryNeural`), и, главное, отдаёт **WordBoundary**
(пословные тайминги: смещение в тексте + время в аудио). Уже используется в
голосовом стеке Hermes на VPS — значит проверен и лёгкий по RAM (зовёт MS-эндпоинт,
без локальной модели). 

Фоллбэк: браузерный `speechSynthesis` + `utterance.onboundary` (без сервера), но
качество русских голосов хуже и поддержка границ слов нестабильна.

## Архитектура (edge-tts, серверная синтезация + клиентская синхронизация)

1. **Извлечение текста (клиент, foliate).** Для текущей секции через `text-walker.js`
   собрать плоскую строку + карту «смещение символа → DOM Range». Так слово из
   тайминга edge-tts мапится обратно в конкретное место в документе.

2. **Синтез (сервер, новый роутер `/api/tts`).** POST `{text, voice, rate}` →
   - запустить `edge-tts` (Python-пакет `edge-tts`), собрать аудио (mp3/opus) и
     список границ слов `[{offset, length, audio_ms}]` из WordBoundary;
   - кэшировать на диск по ключу `sha1(text)+voice+rate` (повторно не синтезировать);
   - вернуть `{audio_url, words: [...]}`.
   RAM-дёшево; на VPS укладывается. Синтез — по секциям/главам, не всю книгу разом.

3. **Воспроизведение + подсветка (клиент).**
   - `<audio>` играет аудио; по `timeupdate`/таймеру сверяем `currentTime` с `audio_ms`
     слов → текущее слово;
   - подсветка через **CSS Custom Highlight API** (`CSS.highlights` + `::highlight()`)
     — без мутации DOM книги, работает и в iframe; либо через foliate `overlayer`;
   - автопрокрутка/листание, чтобы текущее слово было видно (`view.goToFraction`/
     scrollToAnchor по Range);
   - в конце секции — подгрузить следующую и продолжить.

4. **Управление:** play/pause, скорость (rate edge-tts), выбор голоса
   (Svetlana/Dmitry), стоп. Кнопка «▶ Слушать» рядом с «Aa».

## Этапы реализации
1. Бэкенд `/api/tts`: обёртка edge-tts (subprocess/lib), кэш аудио+таймингов, отдача.
2. Клиент: модуль `tts.js` — извлечение текста+карты Range (на базе foliate
   `text-walker`), запрос синтеза, плеер, подсветка слова (CSS Highlight API), автоскролл.
3. UI: кнопка/панель (play/pause/скорость/голос).
4. Edge-cases: длинные секции (резать на части ≤ ~5–10 мин аудио), пауза/seek,
   переход между секциями, очистка подсветки.

## Зависимости
- Python: `edge-tts`.
- Кэш аудио: `data/tts/`.
- Клиент: CSS Custom Highlight API (Chromium/Edge/Firefox — у пользователя Chrome, ок).
