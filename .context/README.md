# `.context` for `reader`

Канонический слой контекста репозитория для людей и ИИ-агентов.
Рантаймом не импортируется. Объясняет форму системы, инженерные правила и решения,
которые легко упустить, читая только код.

## Порядок чтения

1. `SYSTEM_OVERVIEW.md`
2. `TECH_STACK.md`
3. `ADR_LOG.md`
4. Открывайте один специализированный документ — по области, которую меняете.

## Карта файлов

- `SYSTEM_OVERVIEW.md` — назначение, модули, границы рантайма, основные потоки
- `TECH_STACK.md` — языки, фреймворки, библиотеки, непреложные конвенции
- `ADR_LOG.md` — архитектурные решения, которые не меняют без веской причины
- `DB_SCHEMA.md` — модель хранения (SQLite/SQLModel)
- `API_CONTRACTS.md` — HTTP-роуты и контракты
- `CONFIGURATION.md` — переменные окружения, дефолты, приоритеты
- `USER_STORIES.md` — сквозные сценарии, которые поддерживает приложение
- `DOWNLOADERS.md` — источники, цепочка загрузки, адаптеры, фоллбэк платного, анти-бот
- `READERA_SYNC.md` — формат бэкапа ReadEra, Google Drive/rclone, логика синхронизации
- `MONITORING.md` — аккаунты, фиды подписок, детект обновлений, авто-докачка
- `FRONTEND_READER.md` — интеграция foliate-js, темы, навигация, обложки
- `DEPLOY.md` — VPS, systemd, nginx + SSO, cloudscraper, обновление
- `CODE_STYLE.md` — реализация, комментарии, логирование, правила

## Политика обновления

- Обновляйте `.context` в той же ветке, когда меняется архитектура или контракты.
- Документы — короткие, насыщенные, привязанные к реальному коду.
- Если код и документация расходятся — чините немедленно; не оставляйте рассинхрон.
