# Code Style & Rules

## Язык
- **Комментарии, докстринги и сообщения пользователю — на русском.** Идентификаторы и
  код — обычный английский. Полная орфография (ё, кавычки «»).
- Сообщения об ошибках API (`HTTPException`, `RuntimeError` в адаптерах) — по-русски,
  они доходят до UI (статус-строки).

## Backend
- Блокирующий I/O (subprocess, сеть, calibredb) — в синхронных эндпоинтах (FastAPI
  threadpool) или subprocess; **никогда не блокировать event loop**.
- Внешние вызовы (адаптеры, rclone, calibredb) — **best-effort и не валят основной
  поток**: оборачивать в try/except, логировать, возвращать None/пусто. Скачивание
  не должно падать из-за обложки/Calibre/ReadEra.
- Ретраи на флакающих сайтах (author.today, readli) — с паузой; пустой ответ
  (троттлинг) тоже ретраить.
- Тяжёлые импорты (cloudscraper, ebooklib, fanficfare) — по месту/лениво, где уместно;
  межпакетные импорты `app`↔`downloaders` делать **внутри функций** во избежание циклов
  (например, `from ..app import covers` внутри адаптера).
- Секреты не логировать и не отдавать в API. Пароли — только Fernet в БД.

## SQLite
- Все долгие операции (мониторинг) совместимы с **WAL + busy_timeout** (см. `db/session.py`).
  Не убирать прагмы. Коммитить часто и коротко (мониторинг коммитит по работе), не держать
  транзакцию открытой через сетевой I/O без необходимости.

## Frontend
- Без фреймворка и сборки. ES-модули. Настройки — `localStorage` (`reader.prefs`).
- Цвета темы для содержимого книги подставлять **литералами** в `setStyles` (внутри
  iframe CSS-переменных хоста нет).
- foliate shadow DOM closed — внутренние элементы не достать; правки вёрстки рендера —
  в вендоренном `paginator.js`/`view.js` (помечать патчи комментарием-`ВАЖНО`).

## Файлы и идентичность
- Книга в хранилище — `data/books/<sha1>.<ext>`; обложка — `data/covers/<sha1>.<ext>`.
- Дедуп — по `source_url` или нормализованным (title, author); оставлять самый
  полный файл (по размеру). Не плодить Work на одну книгу.

## Документация
- Менять `.context` в той же ветке при изменении архитектуры/контрактов.
- При расхождении кода и доков — чинить сразу.
- Не дублировать в `.context` то, что и так в коде; объяснять «почему» и неочевидное.
